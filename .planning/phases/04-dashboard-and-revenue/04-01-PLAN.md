**1. `getDashboardLessons(tutorId: string)`**

Returns `{ label: string; lessons: Array<{ id: string; scheduled_at: string; status: string; students: { name: string } | null }> }`.

Implementation per D-01, D-04, D-05, D-07:
- Call `getTutorProfile(tutorId)` to get timezone. Default to `'Australia/Sydney'` if null.
- Compute today's boundaries using timezone-aware conversion:
  ```typescript
  import { startOfDay, endOfDay, addDays } from 'date-fns'
  import { toZonedTime, fromZonedTime } from 'date-fns-tz'

  const tz = tutor.timezone ?? 'Australia/Sydney'
  const nowInTz = toZonedTime(new Date(), tz)
  const dayStart = fromZonedTime(startOfDay(nowInTz), tz)
  const dayEnd = fromZonedTime(endOfDay(nowInTz), tz)
  ```
- Query lessons table: `.from('lessons').select('id, scheduled_at, status, students(name)').eq('tutor_id', tutorId).gte('scheduled_at', dayStart.toISOString()).lte('scheduled_at', dayEnd.toISOString()).order('scheduled_at')`
- If results are non-empty, return `{ label: 'Today', lessons: data }`.
- If empty (per D-04), look ahead up to 30 days:
  ```typescript
  const lookaheadEnd = fromZonedTime(endOfDay(addDays(nowInTz, 30)), tz)
  ```
  Query lessons between `dayEnd` and `lookaheadEnd`, ordered by `scheduled_at`.
  If upcoming results exist, take the first lesson's `scheduled_at`, compute its day boundaries, filter to just that day's lessons. Format label as `'Next up: ' + format(toZonedTime(firstDate, tz), 'EEEE d MMM')` using `date-fns-tz` format (e.g., "Next up: Wednesday 26 Mar").
  If no upcoming lessons either, return `{ label: 'Today', lessons: [] }`.

**2. `getDashboardStats(tutorId: string)`**

Returns `{ weeklyLessonCount: number; unpaidCount: number; unpaidTotal: number; monthlyRevenue: number; yearlyRevenue: number }`.

Implementation per D-02:
- Get tutor timezone via `getTutorProfile(tutorId)`.
- Compute timezone-aware week, month, and year boundaries:
  ```typescript
  import { startOfWeek, endOfWeek, startOfMonth, endOfMonth, startOfYear, endOfYear } from 'date-fns'

  const nowInTz = toZonedTime(new Date(), tz)
  const weekStart = fromZonedTime(startOfWeek(nowInTz, { weekStartsOn: 1 }), tz)
  const weekEnd = fromZonedTime(endOfWeek(nowInTz, { weekStartsOn: 1 }), tz)
  const monthStart = fromZonedTime(startOfMonth(nowInTz), tz)
  const monthEnd = fromZonedTime(endOfMonth(nowInTz), tz)
  const yearStart = fromZonedTime(startOfYear(nowInTz), tz)
  const yearEnd = fromZonedTime(endOfYear(nowInTz), tz)
  ```
- Run 4 parallel queries via `Promise.all`:
  1. Lessons this week: `.from('lessons').select('id').eq('tutor_id', tutorId).gte('scheduled_at', weekStart.toISOString()).lte('scheduled_at', weekEnd.toISOString())`
  2. Unpaid invoices: `.from('invoices').select('id, total').eq('tutor_id', tutorId).in('status', ['draft', 'sent'])`
  3. Paid invoices this month: `.from('invoices').select('total').eq('tutor_id', tutorId).eq('status', 'paid').gte('paid_date', format(monthStart, 'yyyy-MM-dd')).lte('paid_date', format(monthEnd, 'yyyy-MM-dd'))` (use `date-fns` format for date-only comparison since `paid_date` is a DATE column)
  4. Paid invoices this year: `.from('invoices').select('total').eq('tutor_id', tutorId).eq('status', 'paid').gte('paid_date', format(yearStart, 'yyyy-MM-dd')).lte('paid_date', format(yearEnd, 'yyyy-MM-dd'))`
- Aggregate in TypeScript:
  ```typescript
  weeklyLessonCount: weekLessons.data?.length ?? 0
  unpaidCount: unpaidInvoices.data?.length ?? 0
  unpaidTotal: unpaidInvoices.data?.reduce((sum, inv) => sum + Number(inv.total), 0) ?? 0
  monthlyRevenue: monthPaid.data?.reduce((sum, inv) => sum + Number(inv.total), 0) ?? 0
  yearlyRevenue: yearPaid.data?.reduce((sum, inv) => sum + Number(inv.total), 0) ?? 0
  ```
- Throw on any Supabase error (same pattern as existing queries).

CRITICAL: Use `fromZonedTime` (NOT `toZonedTime`) to convert local boundaries back to UTC for DB queries. `toZonedTime` is for display only.
CRITICAL: Use `format(date, 'yyyy-MM-dd')` from `date-fns` (not `date-fns-tz`) for paid_date comparisons since paid_date is stored as a DATE column, not TIMESTAMPTZ.

**1. `getMonthlyRevenue(tutorId: string, year: number): Promise<MonthlyRow[]>`**

Implementation per D-08:
- Run two parallel Supabase queries:
  1. All invoices for the year: `.from('invoices').select('id, issued_date, paid_date, total, status').eq('tutor_id', tutorId).gte('issued_date', `${year}-01-01`).lte('issued_date', `${year}-12-31`)`
  2. All completed lessons for the year: `.from('lessons').select('id, scheduled_at, duration_minutes').eq('tutor_id', tutorId).eq('status', 'completed').gte('scheduled_at', `${year}-01-01T00:00:00Z`).lte('scheduled_at', `${year}-12-31T23:59:59Z`)`
- Initialize a Map with 12 entries keyed "YYYY-01" through "YYYY-12", each with zeroed values.
- Group lessons by month of `scheduled_at` (use `.slice(0, 7)` on ISO string for month key). Accumulate `lessonsDelivered` (count) and `hoursTaught` (sum of `duration_minutes / 60`).
- Group invoices by month of `issued_date` (`.slice(0, 7)`). For each invoice:
  - Add `Number(inv.total)` to `amountInvoiced` if `status` is `'sent'` or `'paid'` (drafts excluded per research recommendation).
  - Add `Number(inv.total)` to `amountPaid` if `status === 'paid'` AND `paid_date` is in the same month bucket (group by `paid_date.slice(0, 7)` for paid attribution).
- Compute `outstanding` per month = `amountInvoiced - amountPaid`.
- Convert Map to sorted array of MonthlyRow. Month labels: use `date-fns` `format(new Date(year, monthIndex, 1), 'MMM')` for short month names.

**2. `getStudentRevenue(tutorId: string, year: number): Promise<StudentRow[]>`**

Implementation per D-10:
- Run two parallel Supabase queries:
  1. All invoices for the year with student info: `.from('invoices').select('id, total, status, student_id, students(name)').eq('tutor_id', tutorId).gte('issued_date', `${year}-01-01`).lte('issued_date', `${year}-12-31`)`
  2. All completed lessons for the year with student info: `.from('lessons').select('id, student_id').eq('tutor_id', tutorId).eq('status', 'completed').gte('scheduled_at', `${year}-01-01T00:00:00Z`).lte('scheduled_at', `${year}-12-31T23:59:59Z`)`
- Build a Map keyed by `student_id`. For each student accumulate:
  - `name`: from `students.name` on the invoice/lesson row
  - `lessons`: count of completed lessons
  - `invoiced`: sum of `total` from invoices where `status IN ('sent', 'paid')`
  - `paid`: sum of `total` from invoices where `status === 'paid'`
  - `outstanding`: `invoiced - paid`
- Convert Map to array sorted by `invoiced` descending (default per D-11: most revenue first).
- Return the sorted array.

CRITICAL: Exclude draft invoices from "amount invoiced" â€” only `sent` and `paid` invoices represent billed work.
CRITICAL: `paid_date` attribution for monthly paid amounts â€” a paid invoice may have `issued_date` in March but `paid_date` in April. The "amount paid" column should attribute to the month of `paid_date`, not `issued_date`.
