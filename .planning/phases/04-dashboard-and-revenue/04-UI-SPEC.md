## Spacing Scale

Declared values (all multiples of 4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, gap between metric card stat and label |
| md | 16px | Default element spacing, `p-4` page padding on mobile |
| lg | 24px | Section padding, `p-6` page padding on desktop, gap between dashboard sections |
| xl | 32px | Layout gaps between major sections (today's lessons → metric cards) |
| 2xl | 48px | Major section breaks (between monthly table and per-student table on revenue page) |
| 3xl | 64px | Page-level spacing (not expected to be needed this phase) |

Exceptions:
- Touch targets: minimum 44×44px for all tappable elements (CLAUDE.md constraint). Quick action buttons must be at least `h-11` (44px). Sortable column headers must have `min-h-[44px]` or equivalent padding.
- Bottom navigation clearance: existing `pb-16 md:pb-0` on `<main>` in app layout. Quick actions bar placed as last block in dashboard page content — NOT `sticky bottom-0` — to avoid overlapping bottom nav. (See RESEARCH.md Pitfall 4.)

## Color

All values from `globals.css` CSS variables. This phase uses only established semantic tokens — no new colors introduced.

| Role | CSS Variable | Approximate Value | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) — white | Page background, scrollable surfaces |
| Secondary (30%) | `--card` / `--secondary` | oklch(1 0 0) / oklch(0.97 0 0) — off-white | Metric cards, today's lesson list container, revenue table rows |
| Accent (10%) | `--primary` | oklch(0.205 0 0) — near-black | See accent reserved list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) — red | Not used in this phase — no destructive actions |

Accent (`--primary`) reserved for:
1. Quick action button text and border (outline variant with primary ring on focus)
2. Sortable column header active sort indicator (ChevronUp/ChevronDown icon color when column is sorted)
3. Year navigation chevron buttons on revenue page
4. Link text on lesson rows that deep-link to schedule (hover state only — underline + primary color)

Muted colors used for supporting text:
- `text-muted-foreground` for metric card sub-labels (e.g., "lessons this week"), column headers in revenue tables, empty state body text, "Next up:" date label prefix

Status-specific badge colors follow existing badge variants (no new colors):
- Completed lessons: `variant="secondary"` (neutral grey)
- Scheduled lessons: `variant="outline"` (bordered neutral)

### Tables

Both revenue tables use `overflow-x-auto` wrapper for mobile horizontal scroll. No truncation of numeric values — tables scroll rather than truncate numbers.

Desktop (`md:`): all columns visible inline.
Mobile: horizontal scroll. Sticky first column (`sticky left-0 bg-background`) for student name / month label so users retain context while scrolling right.

Column header tap targets: `<button>` elements with `py-3 px-2 text-left text-sm font-medium text-muted-foreground min-h-[44px]` for sort headers. Non-sortable headers use `<th>` with same padding.

## Interaction Contracts

### Dashboard Lesson Tap (D-06)

Each lesson row in TodayLessons is a `<Link>` navigating to `/schedule?week=[lesson-week-monday]&lesson=[lesson-id]`. The week param is the ISO date of the Monday of the lesson's week (format: `yyyy-MM-dd`). This ensures the schedule page loads the correct week before auto-opening the lesson panel.

Tap target: full row, `min-h-[44px]`, `flex items-center` layout. No status action controls on dashboard.

### Sortable Revenue Table Headers (D-11)

- Each sortable column header is a `<button>` element.
- Default sort: `invoiced` column, descending (most revenue first) — matches D-11.
- Clicking an active sort column reverses direction (asc ↔ desc).
- Clicking a new column sets it to descending first.
- Sort indicator: `ChevronDown` shown when descending, `ChevronUp` when ascending, neither when column inactive. Active column header text uses `text-foreground` (not muted).
- Name column sorts alphabetically (localeCompare).
- All other columns sort numerically.
- Non-interactive rows (D-12): `<tr>` has no click handler, no hover state.

### Year Navigation on Revenue Page (D-09 — Claude's discretion)

- Default year: current calendar year (`new Date().getFullYear()`).
- Year read from `searchParams.year` by Server Component; if absent, defaults to current year.
- Left chevron (`<`) always shown; links to `?year=[year-1]`.
- Right chevron (`>`) shown only if `year < currentYear`; hidden for current year (no future data).
- Navigation is a server-side page reload (Next.js Link), not client state — no `'use client'` needed on the page.

### Loading States

Both pages use `<Suspense>` with skeleton fallback. Skeletons mirror actual layout:
- Dashboard: show 2 skeleton lesson rows + 4 skeleton metric card rectangles.
- Revenue: show 6 skeleton table rows for monthly table; 4 skeleton rows for student table.
- Skeleton style: `bg-muted animate-pulse rounded` blocks at appropriate heights.

## Accessibility Notes

Per CLAUDE.md constraints (WCAG AA):
- All tappable elements: minimum 44×44px touch target.
- Sortable table column headers are `<button>` elements with visible focus ring (`focus-visible:ring-2 focus-visible:ring-ring`).
- Sort state communicated via `aria-sort="ascending"` / `aria-sort="descending"` on `<th>` elements.
- Metric cards use `<dl>` / `<dt>` / `<dd>` semantics or `aria-label` on the card to associate value with label.
- Empty states include an `<h2>` heading so screen readers land on meaningful text.
- Revenue tables include `<caption>` with the current year context: e.g. `<caption className="sr-only">Monthly revenue breakdown for 2026</caption>`.
- Color is never the sole differentiator: sort direction uses both icon shape AND column label weight change.
