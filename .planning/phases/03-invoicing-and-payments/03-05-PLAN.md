<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: End-to-end verification of invoice lifecycle</name>
  <what-built>
Complete invoice lifecycle: creation from student detail, lesson selection, draft/send, PDF preview and download, invoice list with status filtering, mark as paid with receipt, and delete draft.
  </what-built>
  <how-to-verify>
Start the dev server (`npm run dev`) and verify each group:

**1. Invoice creation (INV-01, INV-02, INV-03, INV-04)**
- Navigate to a student with completed lessons
- Click "Create Invoice" button
- Verify: un-invoiced completed lessons shown with checkboxes, all selected
- Uncheck one lesson, verify total updates
- Set issue date and due date, add a note
- Click "Save Draft" — verify redirect to invoice detail, status = draft
- Verify invoice number is INV-0001 format

**2. Invoice list (INV-05)**
- Navigate to /invoices
- Verify the draft invoice appears
- Click status tabs: All, Draft, Sent, Paid, Overdue — each filters correctly
- Verify empty state shows when filtering to a status with no invoices

**3. Invoice detail (INV-06)**
- View the draft invoice detail
- Verify all line items shown with correct dates, durations, rates, amounts
- Verify notes displayed
- Verify total is correct sum

**4. Send invoice (INV-04)**
- On the draft, click "Send Invoice"
- Verify status changes to Sent
- Verify it now appears under Sent tab in list

**5. PDF preview and download (INV-07)**
- On invoice detail, verify PDF preview renders (desktop) or download link shows (mobile)
- Click download — verify PDF opens with correct tutor name, invoice number, dates, bill-to, line items, total, notes

**6. Mark as paid (PAY-01, PAY-02)**
- On the sent invoice, click "Mark as Paid"
- Fill in payment date and method
- Submit — verify toast "Payment recorded"
- Verify status changes to Paid
- Verify "Paid on [date] via [method]" line appears
- Verify receipt info shown on detail page

**7. Delete draft (INV-08)**
- Create another draft invoice
- On detail page, click Delete
- Verify AlertDialog confirmation appears
- Confirm delete — verify redirect to /invoices, invoice gone from list
- Verify the lessons are now available for a new invoice (go back to student, Create Invoice, lessons reappear)

**8. Mobile responsive**
- Narrow viewport to mobile width
- Verify invoice list cards are tappable with adequate size
- Verify create invoice form is usable on mobile
- Verify PDF download link shows instead of preview iframe
  </how-to-verify>
