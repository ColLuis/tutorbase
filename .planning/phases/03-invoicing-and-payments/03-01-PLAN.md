  <action>
**Create `src/lib/queries/invoices.ts`** with these exports:

1. `getUnInvoicedLessons(tutorId: string, studentId: string)` — query lessons where `tutor_id = tutorId`, `student_id = studentId`, `status = 'completed'`, `invoice_id IS NULL`, ordered by `scheduled_at`. Select: id, scheduled_at, duration_minutes, rate, notes, location.

2. `getInvoices(tutorId: string, status?: string)` — query invoices with `students(name)` join. If status is 'overdue': filter `.eq('status', 'sent').lt('due_date', new Date().toISOString().split('T')[0])`. If status is truthy and not 'all': filter `.eq('status', status)`. Order by `created_at` descending.

3. `getInvoice(tutorId: string, invoiceId: string)` — single invoice with joined data: `students(name, parent_name)`, `invoice_items(id, description, quantity, unit_price, amount, lesson_id)`, `receipts(id, receipt_number, paid_at, amount_paid, payment_method)`. Use `.single()`.

4. `getTutorForInvoice(tutorId: string)` — query tutors table for `name` only (per D-07, header shows tutor name only for v1).

**Create `src/lib/actions/invoices.ts`** with `'use server'` directive and these exports:

1. `createInvoice(formData: FormData)` — per D-01 through D-05, INV-01, INV-02, INV-03:
   - Call `verifySession()`
   - Validate with Zod: studentId (uuid), lessonIds (comma-separated string of UUIDs — split and validate), issuedDate (YYYY-MM-DD string), dueDate (YYYY-MM-DD string), notes (optional string), status ('draft' or 'sent')
   - Call `supabase.rpc('get_next_invoice_number')` for invoice_number (INV-02)
   - Calculate subtotal: for each lesson, fetch its rate and duration_minutes, compute `(duration_minutes / 60) * rate`. Sum all amounts.
   - Insert into invoices: tutor_id, student_id, invoice_number, status, issued_date, due_date, subtotal, total (same as subtotal for v1 — no tax), notes
   - Insert invoice_items: for each lesson, one row with description = `"{formatted_date} -- {duration_minutes} min lesson"`, quantity = 1, unit_price = rate, amount = `(duration_minutes / 60) * rate`, lesson_id = lesson.id, tutor_id
   - Update lessons: SET invoice_id = new invoice ID WHERE id IN (lessonIds)
   - `revalidatePath('/invoices')` and `revalidatePath('/students/' + studentId)`
   - Return `{ success: true, invoiceId: newInvoice.id }` or `{ error: string }`

2. `sendInvoice(formData: FormData)` — per D-05, INV-04:
   - Validate invoiceId (uuid)
   - Update invoice SET status = 'sent' WHERE id = invoiceId AND tutor_id AND status = 'draft'
   - revalidatePath, return success/error

3. `deleteInvoice(formData: FormData)` — per INV-08:
   - Validate invoiceId (uuid)
   - Delete invoice WHERE id = invoiceId AND tutor_id AND status = 'draft'
   - The FK ON DELETE SET NULL on lessons.invoice_id handles cleanup automatically
   - ON DELETE CASCADE on invoice_items handles item cleanup
   - revalidatePath, return success/error

4. `markInvoicePaid(formData: FormData)` — per D-13, D-14, PAY-01, PAY-02:
   - Validate invoiceId (uuid), paidDate (YYYY-MM-DD, default today), paymentMethod (optional string)
   - Update invoice SET status = 'paid', paid_date = paidDate
   - Call `supabase.rpc('get_next_receipt_number')` for receipt_number
   - Insert receipt: tutor_id, invoice_id, receipt_number, paid_at = paidDate as TIMESTAMPTZ, amount_paid = invoice.total, payment_method
   - revalidatePath, return success/error

Use zod v4 patterns: `.safeParse()`, `.error.issues[0].message` on failure. Use `createClient()` from `@/lib/supabase/server`. Follow the exact FormData + Object.fromEntries pattern from lessons.ts.
